# FileWork Onboarding Story

> **Purpose**: Define user activation flows
> **Last Updated**: 2026-03-09
> **Owner**: Product Team

---

## First Launch Flow

### Step 1: Welcome Screen
- 显示 FileWork logo 和简介
- "你的本地文件 AI 助手"
- 语言选择 (中文 / English / 日本語)
- [开始使用] 按钮

### Step 2: AI 配置
- 选择 AI 后端:
  - OpenAI (输入 API Key)
  - Anthropic Claude (输入 API Key)
  - Ollama 本地模型 (自动检测)
  - 自定义 API (输入 endpoint + key)
- 选择默认模型
- [测试连接] 验证配置
- [下一步] 按钮

### Step 3: 选择工作目录
- 系统文件选择器打开
- 用户选择一个本地目录
- 显示目录预览 (文件数量、大小、类型分布)
- [确认] 按钮

### Step 4: 第一个任务
- 显示聊天界面
- 预设建议:
  - "帮我整理这个目录的文件"
  - "分析这个目录的内容，生成报告"
  - "找出所有重复的文件"
- 用户输入或选择一个任务
- AI 开始执行，实时显示进度

### Step 5: 完成
- 显示任务结果
- "你已经学会使用 FileWork 了"
- 提示: 可以随时切换工作目录、修改 AI 配置

---

## Activation Metrics

| Metric | Target |
|--------|--------|
| 完成 onboarding | > 80% |
| 执行第一个任务 | > 70% |
| 第二天回访 | > 40% |
| 一周内执行 5+ 任务 | > 25% |

---

## Key Moments

1. **Aha Moment**: 用户看到 AI 自动整理了文件，感受到效率提升
2. **Habit Loop**: 每次打开项目目录时，先用 FileWork 了解目录状态
3. **Value Realization**: 用 FileWork 完成了一个原本需要 30 分钟的手动任务
